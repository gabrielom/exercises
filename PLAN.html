Exercises.

Project plan — a minimalist exercise PWA, hosted on GitHub Pages.

Vision

One small, fast, installable web app for daily physical practice. Four sections — GymStretchingPush-upsPull-ups — presented as a Pinterest-style wall of cards. Every exercise has its own animated figure, a rep counter or a stopwatch (toggleable per exercise), and one-tap set logging. Everything lives in localStorage on your device: no accounts, no servers, works offline.

The app

  • Cards. A fixed, pre-built catalog of 70 exercises. Each card: animated figure (Zdog, ~29 KB, vendored), form cue, reps ⇄ timer toggle, big tap counter or stopwatch, Log set, and today’s totals.
  • Corpo routine. A guided 2-hour stretching & mobility session (details below) with auto-advancing hold/rest timers, chimes, vibration, wake-lock and resume-after-interruption.
  • History. Sets grouped by day with weekly totals. JSON export / import as backup.
  • PWA. Add to home screen → opens standalone, fully offline via a cache-first service worker.

The Corpo routine

Built from the “Corpo” playlist (@gabriel_om, 90 videos, read July 2026), filtered to its stretching & mobility content and focused on three goals: front split (espacate), pancake opening, and feet / ballet work.

Format: 80 slots × (60 s hold + 30 s rest) = exactly 2 hours. Bilateral stretches take one slot per side.

Every completed hold is logged to History automatically. The routine definition lives in one file (js/data.js), so re-deriving it from an updated playlist is a single-file change.

Data

All keys are namespaced (exercises.*) because every *.github.io site shares one localStorage origin.

  • exercises.log — append-only set entries {t, d, ex, mode, v, side?, routine?}
  • exercises.prefs — per-exercise mode overrides
  • exercises.routine — resume state for the Corpo player
  • exercises.settings, exercises.v — theme & schema version

The JSON export is the future sync contract — a companion watch app that counts reps on the fly can produce the same entry shape and merge on import.

Tech

  • Static vanilla ES modules — no framework, no build step. GitHub Pages serves it as-is.
  • Zdog pseudo-3D figures: one parameterized rig + a keyframe pose per exercise; animation runs only for visible cards and freezes under prefers-reduced-motion.
  • Cache-first service worker with a versioned shell — bump VERSION in sw.js to ship updates.

Roadmap

  • Watch companion — count reps automatically (motion sensors), log into the same JSON schema, sync via export/import or Web Share.
  • Playlist re-sync — regenerate the Stretching catalog when “Corpo” changes.
  • Streaks & simple charts in History.
  • Configurable hold/rest durations per block.
